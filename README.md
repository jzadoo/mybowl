 # 점메추 웹 서비스 기획안 — "그릇에 담으면 완성되는 오늘의 메뉴"

## 1. 프로젝트 개요
- **한줄 정의**: 기분·상황·가격대·맛강도·날씨 조건을 그릇에 하나씩 담으면 완성되는, 감각적인 점심 메뉴 추천 웹페이지
- **타겟**: 대학생·사회초년생 등 MZ세대
- **핵심 가치**: 복잡한 필터링 대신 직관적이고 재미있는 인터랙션 + 예쁜 비주얼 + 친구들과 같이 정하는 재미
- **서비스명**: 미정 (예시 후보: "오늘의 그릇", "점메추 한그릇" 등 — 디자인 단계에서 확정)

---

## 2. 핵심 컨셉 — 밥그릇 인터랙션
- 화면 중앙에 빈 그릇 일러스트가 놓여있음
- 5개의 질문(기분 → 상황 → 가격대 → 맛강도 → 날씨)을 한 단계씩 진행
- 각 단계에서 선택한 항목이 "재료"처럼 그릇 위로 톡 떨어지는 마이크로 애니메이션
- 5개를 다 담으면 그릇이 보글보글 끓거나 반짝이는 완성 연출 → 추천 메뉴 카드 등장
- "다시 담기" 버튼으로 처음부터 다시 시도 가능 (직전 추천 메뉴는 제외하고 재계산)

---

## 3. 전체 유저 플로우

### 3-1. 혼자 먹기
1. 랜딩 — "혼자 먹기" / "같이 먹기" 선택
2. 그릇 채우기 (5단계, 각 단계 단일 선택)
3. 완성 연출
4. 결과 카드 — 메뉴명 + 한줄 설명/이미지 + "내 주변에서 찾아보기"(카카오맵 검색 링크) + "다시 담기"

### 3-2. 같이 먹기 (웬투밋 스타일)
1. **방 만들기** — 닉네임 입력 + 참여 인원수 설정 → 초대 링크 발급 (로그인 불필요)
2. **참여** — 링크 클릭 → 닉네임만 입력하고 합류 (앱 설치/가입 없이 바로 가능)
3. **각자 그릇 채우기** — 3-1의 2번과 동일한 5단계, 참여자 각자 자기 몫만 진행
4. **대기 화면** — "3/5명 담는 중" 실시간 표시
5. 설정한 인원이 다 차면 **자동으로 그룹 결과 공개** — 참여자들의 선택 중 항목별로 가장 많이 겹친 조건(다수결)으로 그릇을 다시 채워 추천, 동률이면 랜덤 처리
6. 결과 화면은 혼자 모드와 동일한 카드, 참여자 전원 화면에 동시 노출

---

## 4. 화면 리스트

| 화면 | 설명 |
|---|---|
| 랜딩 | 혼자/같이 모드 선택 |
| 그릇 채우기 (5스텝) | 기분 → 상황 → 가격대 → 맛강도 → 날씨 |
| 완성 연출 | 그릇 완성 애니메이션 |
| 결과 카드 | 메뉴 추천 + 카카오맵 링크 + 다시 담기 |
| 방 만들기 | (같이모드) 닉네임 + 인원수 설정, 링크 발급 |
| 참여 화면 | (같이모드) 닉네임 입력 후 합류 |
| 대기 화면 | (같이모드) 참여 현황 실시간 표시 |
| 그룹 결과 | (같이모드) 전원 동시 노출 |

---

## 5. 입력 조건 5단계 상세

1. **기분/감정**: 스트레스 받음 / 위로받고 싶음 / 신남 / 그냥 그럼 / 든든하게 채우고 싶음
2. **같이 먹는 상황**: 혼밥 / 친구랑 / 회식 / 소개팅·데이트
3. **가격대**: 8천원 이하 / 만원대 / 만오천원 이상
4. **맛 강도**: 자극적으로(매운맛) / 담백하게 / 느끼한거 땡김
5. **날씨**: 자동 감지(선택) 또는 직접 선택 — 더움 / 추움 / 비 / 맑음 (자동 감지 실패 시 수동 선택으로 폴백)

> 선택지 문구·아이콘·이모지는 디자인 단계에서 다듬고, 여기서는 카테고리 뼈대만 고정합니다.

## 6. 데이터 구조 (메뉴 데이터셋)

실시간 식당 API 조회 없이, 태그를 붙인 **큐레이션 JSON 배열**로 관리합니다. (MVP 기준 30~50개 메뉴 권장)

```json
{
  "id": "menu_001",
  "name": "김치찌개",
  "emoji": "🍲",
  "moods": ["위로받고싶음", "든든하게"],
  "situations": ["혼밥", "회식"],
  "price": "8000이하",
  "spicy": "자극적으로",
  "weather": ["추움", "비"],
  "description": "한 마디 소개 문구",
  "kakaoSearchKeyword": "김치찌개 맛집"
}
```

## 7. 추천 로직 (MVP, 심플 태그매칭)

1. 사용자가 선택한 5개 태그와 각 메뉴의 태그를 비교해 일치 개수를 점수로 집계
2. 최고점 메뉴들 중 랜덤 1개 선택 (매번 같은 결과 방지)
3. 동점자가 여럿이면 그중 랜덤
4. "다시 담기" 클릭 시 직전 추천 메뉴는 제외하고 재계산

**그룹 모드**는 참여자 전원의 선택값을 항목별로 최빈값(다수결)으로 합쳐 "가상의 대표 조건"을 만든 뒤 동일 로직을 적용합니다. 항목별 동률은 랜덤 처리합니다.

---

## 8. 같이 먹기 — 방 로직

- 로그인 없이 **닉네임 기반** (닉네임 중복 시 자동으로 구분자 추가, 예: "민지(2)")
- 방장이 참여 인원수를 미리 설정 → 해당 인원이 다 차면 자동으로 결과 계산·공개
- 방 유효시간: 예) 생성 후 30분 미사용 시 만료
- 실시간 동기화가 필요하므로 가벼운 백엔드(Firebase Realtime DB/Firestore 등)가 필요 — 구체 구현 방식은 개발 시 재량

---

## 9. 카카오맵 연동 (MVP는 API 키 없이)

- 결과 카드의 "내 주변에서 찾아보기" 버튼은 카카오맵 장소 검색 딥링크로 연결
  - 예: `https://map.kakao.com/link/search/{키워드}`
- API 키 등록 없이 바로 사용 가능, 클릭 시 새 탭에서 카카오맵 검색 결과로 이동
- **(확장 옵션)** 추후 실제 지도 임베드나 거리순 정렬이 필요해지면 카카오맵 API 키 발급 후 Web API로 고도화 (2026년 7월 21일부로 무료 쿼터 정책이 일부 변경되었으니 개발 전 카카오 디벨로퍼스 문서 확인 권장)

## 10. 디자인 방향 — 미니멀 뉴트로

- **톤앤매너**: 과하지 않은 레트로 감성 + 미니멀한 여백. 그릇·음식 일러스트는 라인아트 또는 도트 텍스처로 통일
- **컬러**: 아이보리/베이지 배경 + 톤다운 포인트 컬러 1~2개 (예: 테라코타, 머스타드, 포레스트그린 중 조합)
- **타이포그래피**: 둥근 산세리프 계열, 헤드라인은 살짝 힘을 준 굵기, 본문은 가볍게
- **인터랙션**: 그릇에 재료가 톡 떨어지는 마이크로 애니메이션, 완성 시 리워드감 있는 연출(파티클/반짝임 등, 과하지 않게)
- **모바일 퍼스트**: 친구랑 같이 볼 때 폰으로 접근할 가능성이 높으므로 모바일 화면 기준으로 우선 설계

---

## 11. 기술 스택 제안

- **프론트**: 반응형 웹페이지. 가벼운 스택 권장 (바닐라 HTML/CSS/JS 또는 경량 프레임워크 중 구현 상황에 맞게 선택)
- **그룹 모드 실시간 동기화**: Firebase(Realtime DB/Firestore) 등 무료 티어로 충분
- **지도 연동**: 카카오맵 딥링크 (API 키 불필요, MVP 기준)
- **로그인/회원가입 시스템**: 없음 (닉네임 기반)
